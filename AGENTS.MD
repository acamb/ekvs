This project use spec driven development.

Always use the docs in `costitution` directory to guide your development:
* mission.md - describes the overall goal of the project
* techstack.md - describes the tech stack and the technical constraints to be used in the project
* roadmap.md - describes the roadmap of the project, including milestones and deadlines

If you are uncertaint about anything, refer to the docs in `costitution` directory for guidance. If you still have questions, ask to the user for clarification.

Each task in the roadmap will be implemented one by one creating a new directory under `features`, which will contain:
* plan.md - describes an ordered list of tasks to be done to implement the feature, with a brief description of each task
* requirements.md - clarifies requirements, scope and decisions. Always update this document as you work on the feature and keep track of user choices if you need to ask the user for clarification.
* validation.md - describes how to validate the implementation of the feature, how to know if it's working as expected and how to test it.
